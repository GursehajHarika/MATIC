Use case

Imagine selling a car on Kijiji or Facebook Marketplace, after agreeing with the buyer on how much they should pay you, the next step is to transfer ownership to the new owner. This is usually done by visiting a governing body which issues and regualtes motor vehicles in your area, in Ontario this can be completed by visiting Service Ontario.

Benefits.

All this sounds relatively normal and is how articles of ownership have been transacted for decades. However, there are a few flaws with this methodology. The first is receiving payment. In the event that one party does not want to pay the agreed amount there is no easy way of stopping the transaction. This is handled wihtin the smart contract. Smart contracts will disallow the transaction if one or both parties do not hold their end of the bargin. The second benefit is an increased level of reliability and speed. Users will no longer need to visit a physical location and are able to transact almost instantly.

NFT...
Now whats that?

the main idea behind an NFT or Non-Fungible-Token is the idea that users can undeniably prove that this user owns this specific NFT and that it has not been modified in anyway shape or form. Ideally, a governing body like the Ministry of Transportation (MTO) would issue NFT's to all motor vehicle owners. Users would then be able to check that this NFT was issued by the MTO and any user that would want to sell their car would be able to without the need of a third-party's approval.

MATIC File System?

Matic File System allows users to transact digital documents by creating an NFT every time a file is uploaded. All files are uploaded to IPFS which is a decentralized file storage system. After succesfully uploading, metadata is extracted to be used for the NFT. After charging the user a small amount in gas fees and service fees the user is issued an NFT, digitally proving that they are the owner of the file.
